Detailed Technical Documentation ‚ñº

1. Frontend & Delivery

The website consists of static HTML, CSS, and JavaScript files hosted on Amazon S3. Content is cached globally via Amazon CloudFront (CDN). Background animations (tsparticles) run client-side, but their data is provided by the backend.

2. Serverless Backend Microservices

The system consists of independent AWS Lambda functions (Microservices) running in a Python 3.12+ environment:

  • Latency Monitor: Measures latency and serves the chart data (via API Gateway).
  • Visitor Counter: Counts page views on every load.
  • Weather Proxy: A dedicated Lambda with a Function URL. This function calls the external OpenWeatherMap API, hiding the API key from the client, and transforms data for background effects.

3. Database (DynamoDB)

Data persistence is handled by NoSQL tables:

  • VisitorCount: Atomic counter that handles concurrent views without write conflicts.
  • UptimeLogs: Time-series latency data. TTL (Time-To-Live) automatically deletes records older than 24 hours to optimize storage costs.

4. Event Scheduling

The Amazon EventBridge Scheduler manages timed tasks. It sends an event every 20 minutes ('Cron Job') to the Monitor Lambda to "wake up" the system and perform measurements, ensuring continuous data collection without manual intervention.

5. CI/CD Automation

Development and deployment are fully automated. A push to the GitHub repository triggers a GitHub Actions workflow that:

  • Syncs static files to S3.
  • Updates Lambda function code in the cloud.
  • Invalidates the CloudFront cache to make changes visible immediately.
