Detailed Technical Documentation ‚ñº

1. Frontend & Delivery

The website consists of static HTML, CSS, and JavaScript files hosted on Amazon S3 (Simple Storage Service). Content is cached and delivered via Amazon CloudFront (CDN) for low-latency global access. Secure connection (HTTPS) is managed by AWS Certificate Manager (ACM).

2. Backend API & Logic

The architecture is fully serverless. All requests are routed through a HTTP API Gateway, which scales automatically. Behind the API endpoints, AWS Lambda functions execute the business logic in a Python 3.13 runtime. These functions only run when triggered.

3. Database & State Management

Data persistence is handled by Amazon DynamoDB (NoSQL). Two main tables are utilized:

  • VisitorCount: Uses atomic counters to increment page views accurately, preventing write conflicts during high traffic.
  • UptimeLogs: Stores latency metrics. TTL (Time-To-Live) is enabled on this table to automatically expire and delete records older than 24 hours.

4. Automated Monitoring

The core of the monitoring system is Amazon EventBridge. It acts as a scheduler, triggering the Monitoring Lambda function every 20 minutes. This function sends HTTP requests to external services, measures latency, and writes logs to DynamoDB. The Frontend fetches this fresh data via the "Reader" Lambda and renders the real-time chart using Chart.js.

5. CI/CD Pipeline (DevOps)

Upon pushing code to the GitHub repository, a GitHub Actions workflow is triggered. This pipeline automatically syncs new HTML/CSS files to S3 and deploys updated Python code to AWS Lambda functions, ensuring zero-downtime updates.
