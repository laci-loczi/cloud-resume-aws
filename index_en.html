Detailed Operational Description

1. Visitor Counting

When the page loads, a request is sent to the ResumeCounter Lambda. This function atomically increments the counter in the VisitorCount DynamoDB table, ensuring no data loss occurs during concurrent visits.

2. Dynamic Background based on Weather

Based on the user's IP address or selection, the PortfolioWeatherAPI Lambda is triggered. It acts as a Proxy: hiding the secret API key, fetching data from OpenWeatherMap, and returning the data to the frontend, which changes the background accordingly.

3. Latency Chart

Two-stage system:

  • Write: EventBridge triggers the PortfolioPinger Lambda every 20 minutes. This "pings" external services (in this case Google, AWS, GitHub, 444), and writes the results into the UptimeLogs table.
  • Read: To draw the chart, the frontend calls the PortfolioStatusAPI Lambda, which reads the last 24 hours of data from DynamoDB.

4. CI/CD Pipeline

System updates are fully automated. After a single git push command, GitHub Actions executes the following steps:

  • S3 Sync: Uploads static files (HTML, CSS, JS, etc.) to the hosting bucket.
  • Lambda Update: Updates the backend Python codes on AWS servers.
  • Invalidation: Clears the CloudFront cache, so changes go live immediately for users.
